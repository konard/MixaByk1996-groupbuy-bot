## 2. Назначение системы

Система представляет собой мультиплатформенный бот для организации совместных закупок с поддержкой:
- Telegram бота
- WebSocket чата в реальном времени

- Интеграции с платёжной системой Точка Банк (сервис Cyclops)

| Переменная | Описание | Где получить |
|------------|----------|--------------|
| `TOCHKA_NOMINAL_ACCOUNT` | Номер номинального счёта | Открывается в Точка Банке |
| `TOCHKA_API_URL` | URL API Cyclops | Pre: `https://pre.tochka.com/api/v1/cyclops`<br>Prod: `https://api.tochka.com/api/v1/cyclops` |
| `TOCHKA_PRIVATE_KEY_PATH` | Путь к приватному ключу | Генерируется самостоятельно |
| `TOCHKA_PUBLIC_KEY_PATH` | Путь к публичному ключу (сертификату) | Генерируется и передаётся в Точку |
| `TOCHKA_PLATFORM_ID` | ID площадки в Cyclops | Выдаётся техподдержкой Точки |

> **Важно**: Cyclops работает только через API, веб-интерфейс не предусмотрен. Для работы необходим расчётный счёт в Точка Банке.

#### О системе Cyclops

Cyclops — это сервис Точка Банка для онлайн-площадок, который позволяет:
- Разделять финансовые потоки между участниками платформы
- Проводить безопасные сделки с использованием номинального счёта
- Выплачивать средства по реквизитам, картам и через СБП (24/7)
- Автоматизировать расчёты с исполнителями (ФЛ, ИП, ЮЛ)

#### Регламент проведения платежей

| Тип платежа | Регламент | Примечания |
|-------------|-----------|------------|
| **По реквизитам** | Пн-Пт: 01:10-19:50 МСК | Вс: 11:10-16:50 (если банк-получатель работает) |
| **Внутри Точки** | 24/7 | Мгновенно |
| **На карту** | 24/7 | До 2 минут, максимум 1 час |
| **СБП (по телефону)** | 24/7 | До 5 минут |

#### Лимиты на переводы

| Тип | Лимит |
|-----|-------|
| **СБП** | До 1 000 000 ₽ за операцию |
| **Карты Mastercard** | 15 000 USD/операция, 50 000 USD/карта/месяц |
| **Карты Visa** | 15 000 USD/операция |
| **Карты МИР** | Без лимитов |
| **ФЛ (технический лимит)** | 400 000 ₽/месяц (по запросу) |

```bash
python3 -c "import secrets; print(secrets.token_urlsafe(50))"
```

### 6.5 Генерация ключей для Cyclops API

Для работы с API Cyclops требуется RSA-ключ. Для каждого слоя (Pre/Prod) нужно генерировать отдельную пару ключей.

```bash
# Генерация приватного ключа (4096 бит)
openssl genpkey -algorithm RSA -out tochka_private.pem -pkeyopt rsa_keygen_bits:4096

# Генерация публичного ключа (сертификата) из приватного
openssl rsa -in tochka_private.pem -pubout -out tochka_public.pem

# Установка прав доступа
chmod 600 tochka_private.pem
chmod 644 tochka_public.pem
```

> **Важно**:
> - Публичный ключ (сертификат) отправляется в техподдержку Cyclops
> - Приватный ключ никогда не передаётся третьим лицам
> - Для Pre и Prod слоёв используются разные пары ключей

## 9. Резервное копирование

### 9.1 Что необходимо бэкапить

| Данные | Частота | Хранение |
|--------|---------|----------|
| База данных PostgreSQL | Ежедневно | 30 дней |
| Загруженные файлы (media) | Еженедельно | 90 дней |
| Конфигурация (.env) | При изменении | Бессрочно |
| SSL сертификаты | При обновлении | Бессрочно |
| Ключи Cyclops API | При изменении | Бессрочно |

### 9.2 Скрипт резервного копирования

```bash
#!/bin/bash
# backup.sh - разместить в /opt/groupbuy/scripts/

BACKUP_DIR="/backups/groupbuy"
DATE=$(date +%Y%m%d_%H%M%S)

# Создать директорию
mkdir -p $BACKUP_DIR

# Бэкап PostgreSQL
docker exec groupbuy-postgres pg_dump -U postgres groupbuy | gzip > $BACKUP_DIR/db_$DATE.sql.gz

# Бэкап .env файла
cp /opt/groupbuy/.env $BACKUP_DIR/env_$DATE.bak

# Удалить бэкапы старше 30 дней
find $BACKUP_DIR -name "*.gz" -mtime +30 -delete
find $BACKUP_DIR -name "*.bak" -mtime +30 -delete

echo "Backup completed: $DATE"
```

### 9.3 Cron задание

```bash
# Добавить в crontab -e
0 3 * * * /opt/groupbuy/scripts/backup.sh >> /var/log/groupbuy-backup.log 2>&1
```

## Приложение C: Интеграция с Cyclops (Точка Банк)

### C.1 Обзор системы Cyclops

**Cyclops** — это сервис Точка Банка для онлайн-площадок, позволяющий разделять финансовые потоки и проводить безопасные сделки между участниками платформы.

#### Ключевые концепции

| Термин | Описание |
|--------|----------|
| **Номинальный счёт** | Счёт площадки для операций с деньгами бенефициаров (ст. 860.1 ГК РФ) |
| **Бенефициар** | Лицо, которому принадлежат деньги на номинальном счёте |
| **Виртуальный счёт** | Техническая сущность для отслеживания баланса бенефициара |
| **Сделка** | Сущность API для указания плательщиков и получателей |
| **Площадка** | Владелец номинального счёта (ваша система) |

### C.2 Требования для подключения

#### Предварительные требования

- [ ] Открыт расчётный счёт в Точка Банке
- [ ] Подготовлены документы для работы (оферта/правила площадки)
- [ ] Заполнен опросник о деятельности
- [ ] Определена схема работы с бенефициарами

#### Документы для согласования

1. **Оферта/Договор площадки** — должен содержать:
   - Указание владельца номинального счёта (полное наименование, ИНН)
   - Указание бенефициара номинального счёта
   - Цель использования номинального счёта
   - Информацию об удержании комиссии (если применимо)
   - Согласие на передачу персональных данных (для ФЛ)

2. **Закрывающие документы** (для сделок):
   - Акты, накладные, УПД
   - Должны содержать: заказчика, исполнителя, объём работ, сумму

### C.3 Процесс подключения

#### Этап 1: Тестовый слой (Pre)

```
URL Pre-слоя: https://pre.tochka.com/api/v1/cyclops
```

1. **Получение доступа**:
   - Передать статический IP-адрес сервера в техподдержку
   - Сгенерировать и передать публичный ключ (сертификат)
   - Получить данные тестовой площадки

2. **Интеграция обязательных методов**:

   **Бенефициары:**

### C.4 Основной бизнес-процесс

```
1. Клиент пополняет номинальный счёт
   (по реквизитам / QR-код СБП / эквайринг)
         ↓
2. Создание бенефициара + загрузка договора оферты
   (если бенефициар ранее не создан)
         ↓
3. Создание виртуального счёта для бенефициара
   (если ранее не создан)
         ↓
4. Идентификация платежа → зачисление на виртуальный счёт
         ↓
5. Создание сделки + загрузка закрывающих документов
         ↓
6. Исполнение сделки → выплаты получателям
```

### C.5 Типы получателей в сделке

| Тип | Описание |
|-----|----------|
| `payment_contract` | Оплата исполнителю по реквизитам |
| `payment_contract_by_sbp` | Оплата через СБП |
| `payment_contract_to_card` | Выплата на карту |
| `commission` | Комиссия площадки (на расчётный счёт) |
| `ndfl` | Бюджетный платёж (НДФЛ) |
| `ndfl_to_virtual_account` | Сбор налогов на виртуальном счёте |

### C.8 Обработка ошибок

| Код | Описание | Рекомендация |
|-----|----------|--------------|
| `403` | Доступ запрещён | Проверить IP, кодировку, подпись |
| `504` | Gateway Timeout | Увеличить таймаут до 60 сек |
| `4408` | Документ не загружен | Дождаться загрузки (до 5 мин) |
| `4436` | Ошибка комплаенс | Платёж не прошёл проверку 115-ФЗ |
| `4953` | Бенефициар обновляется | Подождать между запросами |
