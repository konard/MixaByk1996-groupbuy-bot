## Страницы и их элементы

### Экран авторизации (`LoginModal`)

Полноэкранный оверлей (`position: fixed`, z-index 1100). Две вкладки:

**Вкладка «Вход»:**
- SVG-иконка телефона
- Заголовок «GroupBuy»
- Поле ввода номера телефона (формат `+79001234567`)
- Кнопка «Получить код»
- Ссылка «Зарегистрироваться»

**Вкладка «Регистрация»:**
- SVG-иконка пользователя с плюсом
- Поля: телефон*, email*, имя, фамилия, роль* (покупатель / организатор / поставщик)
- Кнопка «Зарегистрироваться»
- Ссылка «Войти»

**Экран OTP (общий):**
- SVG-иконка конверта с галочкой
- Поле ввода кода (макс. 8 символов, числовая клавиатура)
- Кнопка «Подтвердить»
- Кнопка «Назад»

Флоу: телефон → OTP на email → JWT-токены в localStorage.

---

### Личный кабинет (`Cabinet`)

Многотабовый интерфейс, содержимое зависит от вкладки в Sidebar.

**Верхняя панель (lc-topbar):**
- Аватар пользователя (инициалы на цветном фоне)
- Кнопки действий (зависят от роли):
  - Покупатель: «Создать заявку», «Мои закупки»
  - Организатор: «Создать закупку», «Мои закупки»
  - Поставщик: «Прайс-лист», «Текущие поставки»

**Слайдер категорий:**
- Горизонтальная прокрутка карточек: Подписки, Обмен, Отдых, Жильё, Новости, Блоги
- Каждая карточка — кликабельная, открывает соответствующий раздел

**Профиль:**
- Обложка профиля
- Аватар + имя + роль
- Кнопки быстрых действий (иконки в круге): Кошелёк, Закупки, Чаты, Настройки

**Баланс:**
- Текущий баланс в рублях
- Кнопки «Пополнить» и «Вывести»

**Список закупок:**
- Закупки по статусам (активные, на оплате, завершённые)
- Каждый элемент: название, прогресс, сумма, количество участников

**Чат-панель (двухколоночная):**
- Левая колонка: список чатов закупок (аватар + название + превью)
- Правая колонка: сообщения выбранного чата
- На мобильных — одна колонка со стеком

**Приглашения:**
- Карточки ожидающих приглашений
- Кнопки «Принять» / «Отклонить»

---

### Слайдер закупок (`ProcurementSlider`)

Горизонтальный скролл карточек. Каждая карточка содержит:
- Изображение товара (lazy-loaded) или цветной placeholder
- Название (обрезается до 8 слов)
- Город
- Прогресс-бар (% от целевой суммы)
- Статистика: собрано / цель, дней осталось

Клик открывает `ProcurementModal`.

---

### Модалка закупки (`ProcurementModal`)

Три вкладки:

**«Инфо»:**
- Название, описание, целевая сумма, текущая сумма
- Количество участников
- Бейдж статуса (draft / active / stopped / payment / completed / cancelled)
- Кнопки (зависят от роли и статуса):
  - Покупатель: «Присо��диниться» (сумма + количество + заметки + город), «Покинуть»
  - Организатор: «Остановить», «Утвердить поставщика», «Закрыть закупку»
  - Все: «Открыть чат»

**«Голосование»:**
- Список поставщиков с radio-кнопками
- Текстовое поле для комментария
- Количество голосов за каждого поставщика
- Кнопка «Проголосовать»

**«Участники»:**
- Список участников: аватар, имя, сумма вклада

---

### Чат (`ChatView`)

**Хедер чата:**
- Кнопка «Назад» (возврат в кабинет)
- Аватар закупки + название
- Подзаголовок (количество участников)

**Область сообщений:**
- Разделители по датам («Сегодня», «Вчера», дата)
- Входящие сообщения (белый фон, слева): аватар отправителя, имя, текст, время
- Исходящие сообщения (зелёный фон, справа): текст, время, галочки
- Системные сообщения (по центру)
- Отредактированные сообщения: метка «ред.»
- Удалённые сообщения: «Сообщение удалено»
- Поддержка вложений (файлы, медиа)

**Панель голосования (ChatVotingPanel):**
- Закреплена над полем ввода
- Поставщики + количество голосов
- Кнопка голосования
- Таймер обратного отсчёта при rate limit (429)
- Запросы на закрытие голосования

**Поле ввода:**
- Кнопка прикрепления файла
- Текстовое поле
- Кнопка отправки

**WebSocket-события:**
- Новые сообщения появляются мгновенно
- Редактирование/удаление обновляются на месте
- Индикатор набора текста
- `user_banned` → принудительный logout + редирект на `/banned`

### Sidebar (боковая панель)

Всегда выезжает поверх контента (overlay). Содержит:

**Хедер:**
- Бургер-кнопка (открывает BurgerMenu)
- Заголовок «GroupBuy»
- Кнопка переключения темы (если пользователь не авторизован)

**Поиск:** текстовый инпут для фильтрации закупок по названию/городу.

**Вкладки:** Чаты | Кабинет | Настройки

**Вкладка «Чаты»:**
- Список закупок (аватар, название, превью описания или статистика, время, бейдж непрочитанных)
- Клик → переход в чат

**Вкладка «Кабинет»:** пустая (контент отображается в main-content)

**Вкладка «Настройки»:**
- Карточка профиля (аватар + имя + телефон/email)
- Переключатель темы (Светлая / Тёмная)
- Кнопка «Выйти из аккаунта»

**Футер:** карточка пользователя (клик → настройки)

Полноэкранный оверлей (z-index 200). Закрывается по Escape или клику вне.

**Секция пользователя:** аватар + имя + контакт.

**Пункты меню (зависят от роли):**
- Организатор: «Создать закупку», «Мои закупки», «История»
- Поставщик: «Текущие поставки», «История поставок»
- Покупатель: «Текущие закупки», «Мои заявки», «История»
- Общее: «Подписки»

**Общие пункты:**
- «Активные закупки»
- «Настройки»
- Переключатель «Ночной режим»
- «Выйти»

### Модалки (все остальные)

**Создание закупки (`CreateProcurementModal`):**
- Поля: название, описание (textarea), категория (dropdown), город, целевая сумма, единица измерения, комиссия организатора (%), мин. количество, дедлайн (date picker)
- Категории загружаются из API
- Кнопка «Создать»

**Пополнение баланса (`DepositModal`):**
- Поле суммы (мин. 100₽)
- Выбор метода оплаты: Карта / Электронный кошелёк / Банковский перевод
- Кнопка «Пополнить»

**Вывод средств (`WithdrawModal`):**
- Отображение текущего баланса
- Поле суммы (мин. 100₽, макс. доступный баланс)
- Поле реквизитов (textarea)
- Кнопка «Вывести»

**Прайс-лист (`PriceListModal`):**
- Загрузка файла (XLSX, XLS, CSV, PDF)
- Список популярных товаров (до 20): название + цена
- Кнопки добавления/удаления позиций
- Кнопка «Сохранить»

**Карточка компании (`CompanyCardModal`):**
- Поля: название компании, юр. адрес, почтовый адрес, фактический адрес, ОКВЭД, ОГРН, ИНН, телефон, email
- Валидация обязательных полей
- Кнопка «Сохранить»

**Закрывающие документы (`ClosingDocumentsModal`):**
- Множественная загрузка файлов (PDF, DOC, DOCX, XLS, XLSX)
- Комментарий (textarea)
- Кнопка «Отправить»

**Новость (`NewsModal`):** заголовок + контент (textarea).

**Заявка на закупку (`CreateRequestModal`):** название товара + количество + город + заметки.

### Страницы

- Карточки статистики (StatCard): всего пользователей, новые за сегодня/неделю/месяц

**Пользователи:**
- Фильтры: поиск, роль, платформа, активность, верификация
- Таблица: ID, имя, платформа, роль, статус, баланс
- Действия: вкл/выкл активность, вкл/выкл верификацию, изменить баланс
- Пагинация

**Закупки:**
- Фильтры: статус, категория, город, организатор
- Таблица: ID, название, организатор, статус, участники, сумма
- Действия: обновить статус, переключить «рекомендуемая»

**Платежи:** фильтры (пользователь, статус, тип, даты) + таблица транзакций.

**Категории:** таблица + создание / редактирование / удаление.

**Сообщения:** поиск + модерация (мягкое удаление).

**Остальные страницы:** отчёты, чат с пользователями, лог действий, репутация, жалобы, эскроу, аналитика поиска.

## API-кли��нт (`services/api.js`)

Базовый URL: `/api` (Vite проксирует на `localhost:8000`).

**Механизмы:**
- JWT-токен из localStorage → заголовок `Authorization: Bearer`
- При 401 → автоматический refresh через `/v1/auth/refresh` (одна попытка)
- При 429 → `ApiError` с полем `retryAfter` из заголовка `Retry-After`
- Все запросы регистрируют `AbortController` — отменяются при бане

**Два набора эндпоинтов:**
- Legacy (`/api/users/`, `/api/procurements/`, `/api/chat/`, `/api/payments/`) → Django REST
- v1 (`/api/v1/auth/`, `/api/v1/chat/`, `/api/v1/purchases/`, `/api/v1/voting/`) → NestJS/Go через gateway

## WebSocket (`services/websocket.js`)

Класс `WebSocketManager` �� singleton.

**Подключение:** `ws[s]://{host}/ws/procurement/{id}/?token={jwt}`

**Reconnect:** экспоненциальный backoff (1с → 2с → 4с → ... → 32с), jitter 20%, макс. 10 попыток.

**Дедупликация:** скользящее окно из 500 eventId — дропает повторные сообщения при реконнекте.

**События:**
| Тип | Обработка |
|---|---|
| `message` | Добавление в messages[] |
| `message_edited` | Обновление конкретного сообщения (is_edited: true) |
| `message_deleted` | Пометка как удалённого (is_deleted: true) |
| `typing` | Индикатор набора |
| `user_joined` / `user_left` | Уведомление |
| `vote_cast` / `vote_changed` | Обновление голосования |
| `voting_closed` | Закрытие голосования |
| `user_banned` | **Критический**: закрытие WS, отмена всех fetch, очистка storage, редирект на `/banned` |

## WASM-модуль (`services/wasm.js`)

Rust → WebAssembly. Каждая функция имеет JS-фоллбэк.

**Валидация:** `validatePhone`, `validateEmail`, `validateProcurementForm`

**Форматирование:** `formatCurrency` (рубли), `formatRelativeTime`, `getAvatarColor` (хеш имени → цвет), `getInitials`, `escapeHtml`, `formatMessageText` (ссылки → `<a>`)

**Batch-обработка:** `batchProcessProcurements` (прогресс + дни + валюта за один вызов), `batchProcessMessages`

**Поиск:** `searchProcurements` (fuzzy по title/city/description), `searchMessages`

**Сортировка/статистика:** `sortProcurements`, `aggregateProcurementStats`

## Темизация

Тема определяется при загрузке:
1. Проверяется `localStorage.theme`
2. Если нет — используется `prefers-color-scheme` системы
3. Устанавливается атрибут `data-theme` на `<html>`

Все CSS-переменные меняются через `[data-theme="dark"]` селектор. Переключение — мгновенное, без перезагрузки.

Слушатель `matchMedia('prefers-color-scheme')` от��леживает системные изменения (только если пользователь не выбрал тему вручную).
